Gitは、複数の人が協力してソースコードの変更履歴を追跡・管理するためのバージョン管理システムです。Gitはリポジトリと呼ばれる作業場所を共有し、複数の開発者が変更を追加し、異なる環境で同期するのに非常に役立ちます。この記事では、Git初心者向けに覚えるべきコマンドをまとめてみました。Gitをこれから学びたい方は、ぜひ参考にしてください。

## リポジトリを作成する

Gitを使用するためには、まずローカル環境でリポジトリ（作業ディレクトリ）を作成する必要があります。以下のコマンドを実行して新しいリポジトリを作成します。

```shell
cd <リポジトリを作成したいディレクトリ>
git init
```

このコマンドを実行すると、.gitという隠しディレクトリが作成され、そのディレクトリ内のファイルとフォルダがGitで管理対象となります。

## 変更履歴を追加・登録する

### ファイルを登録
リポジトリに新しいファイルを追加したり、ファイルを変更した場合、次のコマンドで変更を登録できます。

```shell
git add <ファイル名>
```

また、変更が多い場合、以下のコマンドで全ての変更を一括で登録できます。

```shell
git add -A
```

### 変更をコミット
`git add`コマンドを実行しただけでは、変更はまだ正式にコミットされていないため、以下のコマンドで変更をコミットします。

```shell
git commit
```

このコマンドを実行すると、エディタが起動し、コミットメッセージを入力する画面が表示されます。エディタを起動せずにコミットメッセージを直接指定するには、以下のようにコマンドを実行します。

```shell
git commit -m "<コミットメッセージ>"
```

## 既存のリポジトリを取得する

既に存在するリポジトリで作業する場合、リポジトリを自分のローカル環境にコピーする必要があります。以下のコマンドを実行して、リポジトリをクローンします。

```shell
cd <リポジトリをコピーしたいディレクトリ>
git clone <リポジトリのURL>
```

## 差分の取得・変更の反映

### リモートリポジトリの変更を取得
リモートリポジトリに他の開発者が新たな変更をプッシュした場合、以下のコマンドでローカルリポジトリにその変更を取り込みます。

```shell
git pull <リポジトリURL>
```

これにより、他の人が作業した内容を現在のブランチにマージできます。

### 自分の変更をリモートリポジトリに送信
自分の変更をリモートリポジトリにアップロードしたい場合、以下のコマンドを使用します。

```shell
git push <送信先リポジトリURL> <送信先ブランチ>
```

これにより、他のメンバーが変更をプルして取り込むことができます。

## ブランチの操作・管理

### ブランチを作成
新しい機能の実装や緊急の修正など、異なる開発環境を用意したい場合には、ブランチを使用します。以下のコマンドで新しいブランチを作成できます。

```shell
git branch <ブランチ名>
```

### ブランチを切り替える
作業するブランチを切り替えたい場合、以下のコマンドを実行します。

```shell
git checkout <ブランチ名>
```

他に存在しないブランチを指定すると、新しいブランチが自動的に作成されます。

### ブランチを削除
ブランチを削除したい場合、以下のコマンドを使用します。

```shell
git branch -d <ブランチ名>
```

### ブランチの履歴を表示
複数のブランチやタグの分岐履歴を確認するには、以下のコマンドを実行します。

```shell
git show-branch
```

### ブランチのマージ
現在のブランチに別のブランチの変更を取り込むには、以下のコマンドを実行します。

```shell
git merge <変更点の取り込み元ブランチ>
```

## ファイルの状態や履歴を確認する

### 変更されたファイルを表示
追加または変更されたファイ

ルを確認するには、以下のコマンドを実行します。

```shell
git status
```

### コミット履歴を表示
リポジトリへのコミット履歴を確認するには、以下のコマンドを実行します。

```shell
git log
```

### ファイルの差分を表示
特定のファイルが前回のコミットからどのように変更されたのかを確認するには、以下のコマンドを実行します。

```shell
git diff <ファイル名>
```

これらのコマンドを押さえておけば、Gitの基本的な操作ができるようになります。Gitは多くのコマンドが存在しますが、初心者はこれらを覚えておけば実用上の問題はほとんどありません。Gitを学び始める際に役立つガイドとなるでしょう。
